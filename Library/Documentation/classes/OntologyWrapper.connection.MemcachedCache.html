\OntologyWrapper\connectionMemcachedCache

Memcached cache object

This class represents a concrete instance of a {@link CacheObject} implemented using the {@link Memcached} class.

The {@link DSN() DSN} property is represented by the host/port pair and the {@link Connection() connection} by the {@link Memcached} object. A typical string could be memcached://example.net:11211/persistent_id where:

  • memcached: This represents the protocol.
  • example.net: This represents the host.
  • 11211: This represents the port.
  • persistent_id: This represents the persistent ID.

If you want to instantiate a cache that has several hosts or an existing cache, you should instantiate the cache separately with a persistent ID and pass to this class constructor only the persistent ID.

//
// Instantiate cache.
//
$cache = new Memcached( "persistent_id" );
$cache->addServer('mem1.domain.com', 11211, 33);
$cache->addServer('mem2.domain.com', 11211, 67);

//
// In your application code.
//
$my_cache = new MemcachedCache( "memcached:// /persistent_id" );

$sParseURL

$sParseURL : array

URL dictionary.

This static data member holds an array which features as keys the parameters of the {@link parse_url()} function and as values the corresponding {@link Tag} native identifiers that will be used to load the connection parameters into the current object.

The path parameter is missing and should be set in derived classes if used; thew query and fragment elements are handled each by specific methods.

In derived classes this array may be customised without the need to overload the {@link loadMainParameters()} method.

DSN()

DSN(mixed $theValue, boolean $getOld) : mixed

Manage data source name

This method can be used to manage the data source name, it accepts a parameter which represents either the data source name or the requested operation, depending on its value:

The second parameter is a boolean which if TRUE will return the old value when replacing or resetting; if FALSE, it will return the current value.

<h4>Resolve offset</h4>

parseParameters()

parseParameters() : mixed

Parse connection parameters

This method will generate and return a data source name by using the object's connection parameters.

In this class we use:

  • {@link kTAG_CONN_PROTOCOL}: The protocol or scheme.
  • {@link kTAG_CONN_HOST}: The connection host.
  • {@link kTAG_CONN_PORT}: The connection port.
  • {@link kTAG_CONN_USER}: The user code.
  • {@link kTAG_CONN_PASS}: The user password.

Note that the host is required, that is, if the host is missing we set a space in the data source name.

If the resulting data source name is empty, the method will return FALSE.

Derived classes are responsible of managing any other eventual parameter, they may call this method and then add custom parameters to the resulting string.

Returns

mixed —

Data source name or FALSE if empty.

loadMainParameters()

loadMainParameters(\OntologyWrapper\reference $theParameters)

Load connection main parameters

This method will load the parameters parsed from the data source name into the current object, it expects the result of the {@link parse_url()} function and will handle the following parameters:

  • {@link kTAG_CONN_PROTOCOL}: with the scheme.
  • {@link kTAG_CONN_HOST}: with the host.
  • {@link kTAG_CONN_PORT}: with the port.
  • {@link kTAG_CONN_USER}: with the user.
  • {@link kTAG_CONN_PASS}: with the pass.

All parameters are trimmed and eny empty parameter is ignored.

Derived classes are responsible of managing the path part of the URL, they would typically first call the parent method, then handle the path, or they could add the path to the static {@link $sParseURL} array.

The query and fragment elements are handled respectively by {@link loadQueryParameters()} and {@link loadFragParameters()}.

parseDSN()

parseDSN()

Parse data source name

This method will parse the current data source name, extract the connection parameters and set them in the current object parameters.

The method makes use of three methods:

  • {@link loadMainParameters()}: This method will take care of parsing the following parameters:
    • {@link kTAG_CONN_PROTOCOL}: The protocol or scheme.
    • {@link kTAG_CONN_HOST}: The connection host.
    • {@link kTAG_CONN_PORT}: The connection port.
    • {@link kTAG_CONN_USER}: The user code.
    • {@link kTAG_CONN_PASS}: The user password.
    The path parameter should also be parsed in that method, but in this class we do not handle it.
  • {@link loadQueryParameters()}: This method should take care of parsing the query section of the data source name, in this class we ignore these parameters.
  • {@link loadFragParameters()}: This method should take care of parsing the fragment section of the data source name, in this class we ignore this parameter.
