TagCache

Extends \OntologyWrapper\TagCacheObject

Tag cache

This class is a concrete derived instance of the {@link TagCacheObject} which implements a tag cache that uses the {@link Memcached} class.

The class implements the following additional methods:

  • {@link init()}: The method will flush the current cache and reload all tags.
  • {@link Pid()}: The method will return the connection persistent identifier.
  • {@link Connection()}: The method will return the connection resource, which is, in this case, a {@link Memcached} instance.
  • {@link stats()}: The method will return the connection statistics.

This class may be used as a template to implement tag caches that use other cache engines if the {@link Memcached} class is not available.

Methods

Instantiate class.

__construct(mixed $theIdentifier, array $theServers = NULL)

The constructor accepts two parameters:

  • $theId: This string parameter represents the {@limk Memcached} persistent identifier provided to its constructor, the parameter is required, since the cache should be active across sessions.
  • $theServers: This array parameter represents the list of servers that serve the cache, it is equivalent to the parameter of the {@link Memcached::addServers()} method, it is a list of elements comprised by three parameters:
    • Host: The server host.
    • Port: The server port.
    • Weight: The weight of the server relative to the total weight of all the servers in the pool.
    This parameter may be omitted if the cache has been initialised beforehand.

The constructor will first instantiate the {@link Memcached} object, then it will check if the connection resource has already a list of servers associated, if that is the case we assume the cache is already initialised; if that is not the case, we will add the servers provided in the second parameter and load all the current tag identifiers and objects.

Get a tag global identifier

getTagGID(integer $theKey, boolean $doAssert = FALSE) : mixed
inherited

This method should return a tag global identifier given a tag native identifier.

If the provided native identifier is not matched and the second parameter is

TRUE, the method should raise an exception; if the parameter is FALSE, the method should return NULL. If you provide {@link kTAG_NID}, the method will return it. Note that the provided key will be cast to integer, which means that if you did not provide a numeric value, the results will be unexpected.
