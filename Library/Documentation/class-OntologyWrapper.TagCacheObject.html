Class TagCacheObject

Tag cache object

This abstract class is the prototype of classes that implement a tag cache.

The main purpose of a tag cache is to hold in a fast retrieval cache all the current ontology OntologyWrapper\TagObject objects and information, this cache is actively used by all classes derived from OntologyWrapper\OntologyObject for resolving offsets into OntologyWrapper\TagObject references and to cast offset values according to the OntologyWrapper\TagObject declared data type.

Concrete derived classes should implement a constructor that will instantiate the cache and load all current tag identifiers and objects.

The class features the following specific virtual methods:

  • OntologyWrapper\TagCacheObject::init(): This method should initialise the cache.
  • OntologyWrapper\TagCacheObject::setTagId(): This method should set a key/value pair in which the key represents the tag global identifier and the value the tag native identifier.
  • OntologyWrapper\TagCacheObject::getTagId(): This method should return a tag native identifier given a tag global identifier.
  • OntologyWrapper\TagCacheObject::delTagId(): This method should delete a key/value pair in which the key represents the tag global identifier.
  • OntologyWrapper\TagCacheObject::setTagObject(): This method should set a key/value pair in which the key represents the tag native identifier and the value the tag object or array of contents.
  • OntologyWrapper\TagCacheObject::getTagObject(): This method should return a tag object or array of contents given a tag native identifier.
  • OntologyWrapper\TagCacheObject::delTagObject(): This method should delete a key/value pair in which the key represents the tag native identifier.
  • OntologyWrapper\TagCacheObject::getTagGID(): This method should return a tag global identifier given a tag native identifier.

# getTagGID( integer $theKey, boolean $doAssert = FALSE )

Get a tag global identifier

Get a tag global identifier

This method should return a tag global identifier given a tag native identifier.

If the provided native identifier is not matched and the second parameter is TRUE, the method should raise an exception; if the parameter is FALSE, the method should return NULL.

If you provide kTAG_NID, the method will return it.

Note that the provided key will be cast to integer, which means that if you did not provide a numeric value, the results will be unexpected.
