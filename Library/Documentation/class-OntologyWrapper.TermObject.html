Class TermObject

Term object

This class extends OntologyWrapper\OntologyObject to implement a concrete term object class.

A term object holds the necessary information to uniquely identify, document and share a generic term or concept which is not related to a specific context.

For instance, a name is defined as a string or text that identifies something, this is true for both a person name or an object name, however, the term name will bare a different meaning depending on what context it is used in: the term object holds the definition of that will not change with its context.

The class features the following default offsets:

  • kTAG_NID: <em>Native identifier</em>. This required attribute holds the term global identifier. By convention this value is the combination of the namespace, kTAG_NS, and the local identifier, kTAG_LID, separated by the kTOKEN_NAMESPACE_SEPARATOR token. In practice, the global identifier may be manually set. This attribute must be managed with its offset.
  • kTAG_NS: <em>Namespace</em>. This optional attribute is a reference to another term object that represents the namespace of the current term. It is by definition the global identifier of the namespace term. This attribute must be managed with its offset.
  • kTAG_LID: <em>Local identifier</em>. This required attribute is a string that represents the current term unique identifier within its namespace. The combination of the current term's namespace and this attribute form the term's global identifier. This attribute must be managed with its offset.
  • kTAG_LABEL: <em>Label</em>. The label represents the name or short description of the term that the current object defines. All terms <em>should</em> have a label, since this is how human users will be able to identify and select them. This attribute has the kTYPE_KIND_VALUE data type, which is constituted by a list of elements in which the kTAG_SUB_LANGUAGE item holds the label language code and the kTAG_SUB_TEXT holds the label text. To populate and handle labels by language, use the OntologyWrapper\TermObject::Label() offset accessor method. Some terms may not have a language element, for instance the number 2 may not need to be expressed in other ways.
  • kTAG_DEFINITION: <em>Definition</em>. The definition represents the description or extended definition of the term that the current object object defines. The definition is similar to the <em>description</em>, except that while the description provides context specific information, the definition should not. All terms <em>should</em> have a definition, if the object label is not enough to provide a sufficient definition. Definitions have the kTYPE_KIND_VALUE data type in which the kTAG_SUB_LANGUAGE element holds the definition language code and the kTAG_SUB_TEXT holds the definition text. To populate and handle definitions by language, use the OntologyWrapper\TermObject::Definition() offset accessor method.

The OntologyWrapper\TermObject::__toString() method will return the value stored in the native identifier, if set, or the computed global identifier if at least the local identifier is set; if the latter is not set, the method will fail.

Objects of this class can hold any additional attribute that is considered necessary or useful to define and share the current term. In this class we define only those attributes that constitute the core functionality of the object, derived classes will add attributes specific to the domain in which the object will operate.

# __toString( )

Return global identifier

Return global identifier

If the native identifier, kTAG_NID, is set, this method will return its value. If that offset is not yet set, the method will compute the global identifier by concatenating the object's namespace, kTAG_NS, with the object's local identifier, kTAG_LID, separated by the kTOKEN_NAMESPACE_SEPARATOR token. This will only occur if the object has the local identifier, if that is not the case, the method will return an empty string to prevent the method from causing an error.

# Label( string $theLanguage, mixed $theValue = NULL, boolean $getOld = FALSE )

Manage tag label

Manage tag label

This method can be used to add, retrieve and delete individual labels by language, while managing all the labels as a whole can be done using the kTAG_LABEL offset.

The method expects the following parameters:

  • $theLanguage: This parameter holds the language code of the text we want to manage. If NULL, it means that there is a text without a language; this can occur if the label is a name unrelated to any language.
  • $theValue: This parameter identifies the label text or operation:
    • NULL: Return the label of the provided language.
    • FALSE: Delete the label of the provided language.
    • other: Any other value is cast to string and interpreted as the text of the label in the provided language which will be inserted, or that will replace an existing entry.
  • $getOld: Determines what the method will return:
    • TRUE: Return the value before it was eventually modified.
    • FALSE: Return the value after it was eventually modified.

The method will return either the label text, or NULL if the language was not matched.
