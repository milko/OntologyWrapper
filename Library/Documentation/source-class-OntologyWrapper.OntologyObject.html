 11: /**
 12:  * Ontology object
 13:  *
 14:  * The main purpose of this class is to match all offsets to an ontology domain, ensuring
 15:  * that any value held by the embedded object array resolves into an ontology element.
 16:  *
 17:  * The unique identifier of an object is represented by the <i>native identifier offset</i>,
 18:  * <b><tt>{@link kTAG_NID}</tt></b>, which is the only alphabetic offset allowed in this
 19:  * class, all other offsets <i>must</i> be numeric, equivalent to an integer. This is
 20:  * because these numeric constants refer to {@link Tag} instances which define and
 21:  * document the object's data values.
 22:  *
 23:  * It is possible to set an offset by providing a string value: in that case the provided
 24:  * value is interpreted as the tag global identifier which must be resolved into the tag's
 25:  * native identifier.
 26:  *
 27:  * No offset may hold the <tt>NULL</tt> value, setting an offset with this value is
 28:  * equivalent to deleting the offset.
 29:  *
 30:  * Requesting an inexistant offset will not trigger a warning, instead, the <tt>NULL</tt>
 31:  * value will be returned, an indication that the offset doesn't exist.
 32:  *
 33:  *  @author     Milko A. Škofič <m.skofic@cgiar.org>
 34:  *  @version    1.00 10/01/2014
 35:  */
