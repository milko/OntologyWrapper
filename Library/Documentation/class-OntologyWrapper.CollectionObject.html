Class CollectionObject

Collection object

This abstract class is the ancestor of all classes representing database collection instances, this class extends the OntologyWrapper\ConnectionObject class to implement collection specific functionality prototypes.

# insert( reference & $theObject, array $theOptions = Array() )

Insert an object

Insert an object

The method expects the provided parameter to be either an array or an ArrayObject instance.

The method will call the virtual OntologyWrapper\CollectionObject::insertData() method, passing the received object to it, which will perform the actual insert.

The method will return the inserted object's identifier, kTAG_NID.

This method will also take care of setting the kTAG_CLASS offset.

# resolve( mixed $theIdentifier, mixed $theOffset = kTAG_NID, boolean $asObject = TRUE )

Resolve an object

Resolve an object

This method should select an object in the current collection matching the provided native identifier with the provided offset and return either the object, if the second parameter is TRUE, or an array if FALSE; if the object cannot be resolved, the method should return NULL.

If there are more than one objects selected, this method should only return the first.

Concrete derived classes should implement this method.

# setSequenceNumber( string $theSequence, integer $theNumber = 1 )

Set sequence number

Set sequence number

This method should initialise a sequence number associated to the provided parameter. This operation is equivalent to resetting an auto-number for a database.

Once the sequence is set, the next requested sequence number will hold the value set by this method, so to start counting from 1 you should provide this value to this method.

This method is intended to be handled by database objects, in this class we simply let the object's parent, a database, perform the action.

Derived classes should never need to overload this method.

# getSequenceNumber( string $theSequence )

Return sequence number

Return sequence number

This method should return a sequence number associated to the provided parameter. This operation is equivalent to requesting an auto-number for a database.

Each time a sequence number is requested, the sequence seed is updated, so use this method only when the sequence is required.

If the sequence selector is not found, a new one will be created starting with the number 1, so, if you need to start with another number, use the OntologyWrapper\CollectionObject::setSequenceNumber() before.

This method is intended to be handled by database objects, in this class we simply let the object's parent, a database, perform the action.

Derived classes should never need to overload this method.
