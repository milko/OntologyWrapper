Detailed Description

Tag

A tag object is used to identify, document and share a data property, it represents the metadata of a data property, or its definition in an ontology.

This class features a kTAG_TERMS offset which is an array of Term references representing a path of vertex and predicate terms of the ontology graph, this chain of elements represents a lexical construct, like a phrase, that provides the explanation or definition of the current object.

Using a chain of terms, rather than a single term, to provide metadata for a data element gives much more flexibility and reusability when defining a data dictionary. This sequence is divided in three main sections:

  • Feature: The first vertex represents the data element feature or trait, this term defines what the data element is.
  • Method: The vertices between the first and the last ones represent the methodology or method by which the described data was obtained.
  • Scale: The last vertex of the path represents the unit or scale in which the described value is expressed in.

The concatenation of these term references, separated by the kTOKEN_INDEX_SEPARATOR becomes the object global identifier which is stored in the object's native identifier offset.

Tags have another offset, kTAG_ID_SEQUENCE, which is an integer sequence number: this value must be unique within the tags domain of the current ontology. Unlike global identifiers, this value may change across implementations, but this is the value used to uniquely identify tags among the other elements of the ontology and database.

All offsets in all classes, including this one, are tag sequence numbers, which makes the Tag class key in the structure and behaviour of all the elements implemented in this library.

The class features the following default offsets:

  • kTAG_NID: Native identifier. This required attribute holds a string value which represents the global identifier of the current object. This identifier is immutable and represents the unique key. This string is constituted by the concatenation of all term references stored in the current object's branch, kTAG_TERMS. This attribute must be managed with its offset; in derived classes it will be automatically assigned end only available as read-only.
  • kTAG_ID_SEQUENCE: Sequence. This required attribute holds an integer value which represents the current object's sequence number, as with the global identifier, this value must be unique, except that it may change across implementations. All offset keys in all objects derived from this class ancestor are references to this sequence number. This attribute must be managed with its offset; in derived classes it will be automatically assigned end only available as read-only.
  • kTAG_TERMS: Branch. This required attribute holds the list of terms comprising the current tag: this is an array of term references provided as an odd sequence of vertices and predicates forming a path of the ontology graph in which the first vertex defines the feature, the middle ones define the methodology and the last element indicates the scale or unit. To populate the elements of this path, use the TermPush() amd TermPop() offset accessor methods which respectively add and remove elements of the branch as if it was a stack.
  • kTAG_DATA_TYPE: Data type. This attribute is an enumerated set of values listing all the data types that the value of the property defined by the current tag may take. To populate and handle individual data types use the DataType() offset accessor method. This property is required by all tag objects.
  • kTAG_DATA_KIND: Data kind. This attribute is an enumerated set of values providing the data attributes of the property defined by the current tag. This may be whether the property is a list of values, or if the property is required or not. To populate and handle individual data kinds use the DataKind() offset accessor method.
  • kTAG_LABEL: Label. The label represents the name or short description of the data property that the current object defines. All tags should have a label, since this is how human users will be able to identify and select them. This attribute has the kTYPE_LANGUAGE_STRINGS data type, which is constituted by a list of elements in which the kTAG_LANGUAGE item holdsthe label language code and the kTAG_TEXT holds the label text. To populate and handle labels by language, use the Label() offset accessor method.
  • kTAG_DESCRIPTION: Description. The description tag represents the description or extended definition of the property that the current tag defines. The description is similar to the definition, except that while the definition provides a description of the object it defines unrelated to context, the description adds to the definition the elements added by the current context. All tags should have a description, if the tag label is not enough to provide a sufficient description or definition. Descriptions have the kTYPE_LANGUAGE_STRINGS data type, which is constituted by a list of elements in which the kTAG_LANGUAGE item holds the description language code and the kTAG_TEXT holds the description text. To populate and handle descriptions by language, use the Description() offset accessor method.

The __toString() method will return the value stored in the global identifier, if set, or the computed global identifier which is the concatenation of all term references stored in the kTAG_TERMS offset, separated by the kTOKEN_INDEX_SEPARATOR.

The class also features a pair of methods, Feature() and Scale(), which respectively return the first and last vertex of the ontology branch this object represents: the feature is a term that represents the main feature or trait that the current tag defines, the scale is a term that represents the scale or unit in which values defined by the current tag are expressed in.

Objects of this class can hold any additional attribute that is considered necessary or useful to define and share the current tag. In this class we define only those attributes that constitute the core functionality of the object, derived classes will add attributes specific to the domain in which the object will operate and eventually restrict the functionality of other offsets to provide referential integrity.

The object is considered initialised, isInited(), if it has at least the terms path, kTAG_TERMS, with an odd number of elements, the data type, kTAG_DATA_TYPE, and the label, kTAG_LABEL.

OntologyWrapper\Tag::__toString ( )

Return global identifier

The global identifier of tags is stored in its kTAG_NID offset: if set, this method will return that value. If that offset is not set, the method will concatenate the value of all the elements of the kTAG_TERMS separated by the kTOKEN_INDEX_SEPARATOR token.

If the kTAG_TERMS offset is not set, the method will return an empty string.
