Detailed Description

Persistent object

This abstract class is the ancestor of all classes representing objects that can persist in a container and that are constituted by ontology offsets.

The main purpose of this class is to add the status and persistence traits providing the prototypes needed to implement concrete persistent objects.

The class makes use of the Status and Persistence traits:

  • Status: This class handles a bitfirld data member that keeps track of the object's status:
    • isDirty(): This flag is set whenever any offset is modified, this status can be tested whenever the object should be stored in a persistent container: if set, it means the object has been modified, if not set, it means that the object is identical to the persistent copy.
    • isCommitted(): This flag is set whenever the object has been loaded or stored into a persistent container. This status can be useful to lock properties that cannot change once the object is stored.
  • Persistence: This class handles the object persistence.

Objects derived from this class must define a constant called kSEQ_NAME which provides a <em<string representing the default collection name for the current object: methods that commit or read objects of a specific class can then resolve the collection given a database.

OntologyWrapper\PersistentObject::__construct (   $theContainer = NULL,
  $theIdentifier = NULL 
)

Instantiate class.

Objects derived from this class share the same constructor prototype, they should not overload this method. The method accepts two parameters:

  • $theContainer: This may either be an array containing the object's persistent attributes, or a reference to a Wrapper object. If this parameter is NULL, the next parameter will be ignored.
  • $theIdentifier: This parameter represents the object identifier or the object persistent attributes: in the first case it will used to select the object from the provided container, in the second case, it is assumed that the provided array holds the persistent attributes of an object committed in the provided container.

The workflow is as follows:

  • Empty object: Both parameters are omitted.
  • Filled non committed object: The first parameter is an array.
  • Load object from container: The first parameter is a Wrapper object and the second parameter is a scalar identifier.
  • Filled committed object: The first parameter is Wrapper object and the second parameter is an array holding the object's persistent data.

Any other combination will raise an exception.

This constructor sets the committed flag, derived classes should first call the parent constructor, then they should set the inited flag.

OntologyWrapper\PersistentObject::commit ( Wrapper  $theWrapper)

Insert the object

This method should commit the current object into the provided persistent store.

In this method we perform the following steps:

  • We resolve the eventually provided persistent store into a collection object, or we use the current object's collection; if this is not set, or if the collection canot be resolved, the method will raise an exception.
  • We call the preCommit() method that is responsible for preparing the object for being committed.
  • If the object is not ready, isReady(), we raise an exception.
  • We pass the current object to the collection's commit method and recuperate the identifier.
  • We call the postCommit() method that is responsible of cleaning up the objecxt after the commit.
  • We return the object's identifier.

If any of the above steps fail the method must raise an exception.

OntologyWrapper\PersistentObject::postCommit (   $theOperation = 0x00)
protected

Cleanup object after commit

This method should cleanup the object after it was committed, it should perform eventual identifiers and commit the eventual related objects.

The method accepts a single bitfield parameter that indicates the current operation:

  • 0x01: Insert.
  • 0x11: Update.
  • 0x10: Delete.

The first bit is set if the object is committed and the second bit is set if we are storing the object.

In this class we do nothing.

OntologyWrapper\PersistentObject::preCommit (   $theOperation = 0x00)
protected

Prepare object for commit

This method should prepare the object for being committed, it should compute the eventual identifiers and commit the eventual related objects.

The method accepts a single bitfield parameter that indicates the current operation: the first bit is set if the object is committed and the second bit is set if we are storing the object.

  • 0x01: Insert

    • Check if the object is isInited().
    • : Validate object. : Set object identifiers. : Check if object is ready. : Commit related objects.
    • 0x11: Update
      • Nothing yet (this operation should not be implemented).
    • 0x10: Delete
      • Check if the object has its native identifier, kTAG_NID.

    Derived classes should not overload this method, they should, instead, overload the called methods.

Resolve the collection

This method should return a CollectionObject instance corresponding to the persistent store in which the current object was either read or will be inserted.

The method expects the object to feature a constant, kSEQ_NAME, which serves the double purpose of providing the default collection name and the eventual sequence number index: the method will use this constant and the provided database reference to return the default CollectionObject instance.
