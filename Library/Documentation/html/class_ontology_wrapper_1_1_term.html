Detailed Description

Term

A term object holds the necessary information to uniquely identify, document and share a generic term or concept which is not related to a specific context.

For instance, a name is defined as a string or text that identifies something, this is true for both a person name or an object name, however, the term name will bare a different meaning depending on what context it is used in: the term object holds the definition of that will not change with its context.

The class features the following default offsets:

  • kTAG_NID: Native identifier. This required attribute holds the term global identifier. By convention this value is the combination of the namespace, kTAG_NAMESPACE, and the local identifier, kTAG_ID_LOCAL, separated by the kTOKEN_NAMESPACE_SEPARATOR token. In practice, the global identifier may be manually set. This attribute must be managed with its offset.
  • kTAG_NAMESPACE: Namespace. This optional attribute is a reference to another term object that represents the namespace of the current term. It is by definition the global identifier of the namespace term. This attribute must be managed with its offset.
  • kTAG_ID_LOCAL: Local identifier. This required attribute is a string that represents the current term unique identifier within its namespace. The combination of the current term's namespace and this attribute form the term's global identifier. This attribute must be managed with its offset.
  • kTAG_LABEL: Label. The label represents the name or short description of the term that the current object defines. All terms should have a label, since this is how human users will be able to identify and select them. This attribute has the kTYPE_LANGUAGE_STRINGS data type, which is constituted by a list of elements in which the kTAG_LANGUAGE item holds the label language code and the kTAG_TEXT holds the label text. To populate and handle labels by language, use the Label() offset accessor method. Some terms may not have a language element, for instance the number 2 may not need to be expressed in other ways.
  • kTAG_DEFINITION: Definition. The definition represents the description or extended definition of the term that the current object object defines. The definition is similar to the description, except that while the description provides context specific information, the definition should not. All terms should have a definition, if the object label is not enough to provide a sufficient definition. Definitions have the kTYPE_LANGUAGE_STRINGS data type in which the kTAG_LANGUAGE element holds the definition language code and the kTAG_TEXT holds the definition text. To populate and handle definitions by language, use the Definition() offset accessor method.

The __toString() method will return the value stored in the native identifier, if set, or the computed global identifier if at least the local identifier is set; if the latter is not set, the method will fail.

Objects of this class can hold any additional attribute that is considered necessary or useful to define and share the current term. In this class we define only those attributes that constitute the core functionality of the object, derived classes will add attributes specific to the domain in which the object will operate.

The object is considered initialised, isInited(), if it has at least the local identifier, kTAG_ID_LOCAL, and the label, kTAG_LABEL.

Member Function Documentation

OntologyWrapper\Term::__toString ( )

Return global identifier

If the native identifier, kTAG_NID, is set, this method will return its value. If that offset is not yet set, the method will compute the global identifier by concatenating the object's namespace, kTAG_NAMESPACE, with the object's local identifier, kTAG_ID_LOCAL, separated by the kTOKEN_NAMESPACE_SEPARATOR token. This will only occur if the object has the local identifier, if that is not the case, the method will return an empty string to prevent the method from causing an error.

static OntologyWrapper\Term::DefaultOffsets ( )
static

Return default offsets

This method will return the current object list of default offsets, these offsets represent the default offsets of the object, which means that all objects derived from this class may feature these offsets. This method is used to exclude these offsets from statistical procedures, such as CollectOffsets(), since it is implied that these offsets will be there.

In this class we return an empty array.

array List of default offsets.
