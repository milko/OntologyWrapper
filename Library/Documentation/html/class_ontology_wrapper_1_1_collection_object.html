Detailed Description

Collection object

This abstract class is the ancestor of all classes representing database collection instances, this class extends the ConnectionObject class to implement collection specific functionality prototypes. 

 @author            Milko A. Škofič <m.skofic@cgiar.org>
 @version   1.00 06/02/2014

The method will call the virtual insertData() method, passing the received object to it, which will perform the actual commit.

The method will return the inserted object's identifier, kTAG_NID.

OntologyWrapper\CollectionObject::getSequenceNumber (   $theSequence)

Return sequence number

This method should return a sequence number associated to the provided parameter. This operation is equivalent to requesting an auto-number for a database.

Each time a sequence number is requested, the sequence seed is updated, so use this method only when the sequence is required.

If the sequence selector is not found, a new one will be created starting with the number 1, so, if you need to start with another number, use the setSequenceNumber() before.

This method is intended to be handled by database objects, in this class we simply let the object's parent, a database, perform the action.

Derived classes should never need to overload this method.

abstract

Resolve an object

This method should select an object in the current collection matching the provided identifier with the provided offset.

The method expects the following parameters:

  • $theIdentifier: This parameter represents the value to match with the provided offset.
  • $theOffset: This parameter represents either the native or persistent identifier of the offset tag.
  • $asObject: This parameter determines what the method should return:
    • TRUE: Return the object; if there are more than one objects selected, this method should only return the first.
    • FALSE: Return the object array; if there are more than one objects selected, this method should only return the first.
    • NULL: Return the objects count.

Concrete derived classes should implement this method.

OntologyWrapper\CollectionObject::setSequenceNumber (   $theSequence,
  $theNumber = 1 
)

Set sequence number

This method should initialise a sequence number associated to the provided parameter. This operation is equivalent to resetting an auto-number for a database.

Once the sequence is set, the next requested sequence number will hold the value set by this method, so to start counting from 1 you should provide this value to this method.

This method is intended to be handled by database objects, in this class we simply let the object's parent, a database, perform the action.

Derived classes should never need to overload this method.
