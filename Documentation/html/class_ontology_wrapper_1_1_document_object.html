Detailed Description

Document object

This class represents the main building block of this library, it implements a base document object which is an extension and restriction of the base ArrayObject class, the inherited array represents the object data members, while eventual optional object members are only used for internal private matters.

The main purpose of this class is to match all offsets to an ontology domain, ensuring that any value held by the embedded object array resolves into an ontology element.

The unique identifier of an object is represented by the native identifier offset, DocumentObject::kTAG_NID, which is the only alphabetic offset allowed in this class, all other offsets must be numeric, equivalent to an integer. This is because these numeric constants refer to ontology instances which define and document the object's data values.

It is possible to set an offset by providing a string value: in that case the provided value is interpreted as the tag global identifier which must be resolved into the tag's native identifier.

No offset may hold the NULL value, setting an offset with this value is equivalent to deleting the offset.

Requesting an inexistant offset will not trigger a warning, instead, the NULL value will be returned, an indication that the offset doesn't exist.

OntologyWrapper\DocumentObject::offsetGet (   $theOffset)

Return a value at a given offset

This method should return the value corresponding to the provided offset.

We overload this method to implement the object's offset rules:

  • integer: When provided with an integer value, the method will behave as its ancestor.
  • DocumentObject::kTAG_NID: When provided with this value, the method will behave as its ancestor.
  • string: When provided with a string, the method will call the protected _resolveOffset() method that should resolve the string into a numeric offset.

If the provided offset cannot be resolved, or if the resolved offset cannot be found, the method will return NULL.

This method will not generate warnings for non matching offsets.

We overload this method to implement the object's offset rules:
